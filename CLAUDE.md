> 이 문서는 Claude Code가 이 저장소에서 작업할 때 항상 지켜야 할 규칙의 요약본이다.

## 1. 목적 / 범위

Notion 페이지에 임베드하는 개인용 위젯과 교사용 수업 도구를 하나의 사이트로 제공한다.

- **위젯**: 뽀모도로 · 캘린더(표시형 + Notion 할일 DB 연동) · 시계 · 플립 시계 · 날씨 · 수업 타이머 · 자리뽑기 · 랜덤뽑기 · 사다리타기 · 탭 묶음 · 지도(Notion 장소 DB 연동) · 할일 목록(Notion DB 연동) · 주사위 · 화이트보드/블랙보드 · 갤러리(Notion 파일과 미디어 속성 연동)
- **빌더**: 위젯 선택 → 설정 UI → 임베드 URL(`?c=` 설정 문자열) 발급
- **Notion 연동**: 대시보드 외에도 캘린더·할일 목록·지도·갤러리가 읽기 전용으로 연동한다(내용은 응답으로 통과만, 서버 저장·캐시·로그 금지 — §5의 7번 예외 규약과 동일 원칙). 쓰기(기록 append)는 여전히 대시보드/뽀모도로만.
- **제외**: 런타임 LLM, 유튜브 위젯, 마켓플레이스, 유료 결제, cron 배치 동기화, 네이티브 앱/PWA

**성격**: 런타임 LLM이 없는 결정론적 웹앱. "에이전트"는 빌더로서의 Claude Code만 가리킨다. 런타임 판단·검증은 전부 코드(zod 스키마, 순수함수)로 처리한다.

## 3. 아키텍처 — 두 계층과 정적 우선 원칙

```
[정적 계층]  React 위젯 6종 — 설정은 URL/localStorage에서만 읽음. 서버 불필요.
     │ (대시보드/자동기록만 통과)
[연동 계층]  Node+Express — OAuth · 위젯 토큰 · Notion 프록시 · 캐시(TTL 30~60s) · SQLite
     │
[Notion API]  ~3 req/s
```

- **정적 우선**: 대시보드 외 위젯은 어떤 코드 경로에서도 서버에 의존하지 않는다. 빌드 시 import 경계로 강제한다. 서버 장애 시에도 6종은 정상 동작.
- **온디맨드만**: cron 없음. 대시보드가 열릴 때 읽고, 기록이 발생할 때 쓴다.
- **인증 이원화**: 빌더(직접 방문) = OAuth + 퍼스트파티 세션 쿠키. 임베드(iframe) = URL 내 **HMAC 서명 위젯 토큰**(서드파티 쿠키 차단 때문에 쿠키 불가).
- **1차 연동 구현 = 내부 통합 토큰 모드** (주식 위젯과 동일 패턴): 서버 `.env`의 `NOTION_TOKEN`(내부 통합) + 위젯 설정의 DB ID. 공개 OAuth·위젯 토큰은 공개 통합 심사 후 도입(부록 D). 쓰기 멱등키 = 클라이언트 세션 UUID(SQLite `recorded_sessions`).
- **라우팅**: `/` 빌더 홈 · `/edit/:id` 위젯 편집 · `/w/:widget?c=` 임베드(컴팩트) · `/f/:widget?c=` 전체화면(프로젝터용 대형 타이포). 동일 컴포넌트가 레이아웃 모드만 달리 렌더링.
- **빌더 홈 구조** (기존 주식 위젯 사이트 notion-stock-widget.vercel.app 구조 준수): **My Widgets**(만든 위젯 인스턴스 목록, localStorage) + **Explore**(위젯 종류 갤러리 + 카테고리 필터 + 라이브 미리보기). Explore에서 만들기 → `/edit/:id`(설정 폼 + iframe 미리보기 + 임베드/전체화면 링크 복사, 자동 저장).

## 4. 폴더 구조

```
/notion-widget-hub
  ├── CLAUDE.md
  ├── package.json            # npm workspaces
  ├── /.claude/skills          # 빌드 참조 묶음(런타임 스킬 아님) — §10
  ├── /apps/web                # React+Vite: 빌더 + 위젯 + 전체화면
  ├── /apps/server             # Express: OAuth · 위젯토큰 · Notion 프록시 · SQLite
  ├── /packages/core           # 순수함수: 설정 코덱, 뽀모도로 상태머신,
  │                            #   자리배정, 추첨, 대시보드 집계 + zod 스키마
  └── /output                  # 로그 · 쓰기 실패 큐 스냅샷
```

**스택**: TypeScript + React + Vite(웹) · Node + Express + better-sqlite3 + zod(서버) · Vitest(테스트) · 웹 = 정적 배포, 서버 = Railway.

## 5. 도메인 불변식 (위반 금지)

1. **append-only**: 기록 DB에 행 생성만 한다. 기존 행 수정·삭제 절대 금지. 시스템 생성 행도 생성 후 건드리지 않는다.
2. **필드 소유권**: 시스템이 채우는 속성은 매핑된 필수 속성(+ 선택 `출처` 태그)뿐.
3. **집계는 코드, 원본은 Notion**: 통계·진행률은 core 순수함수가 계산해 화면에만 표시. 집계값을 Notion에 역기록하지 않는다.
4. **중복 방지**: 뽀모도로 자동기록 중복키 = 클라이언트 세션 UUID. 동일 UUID 재전송은 멱등 처리.
5. **정적 우선**: 대시보드 외 위젯은 서버 비의존(import 경계로 강제).
6. **시크릿 경계**: Notion 토큰·서명 키는 서버에만. 브라우저에 절대 내려보내지 않는다. 위젯 토큰은 범위 최소화 + 재발급 시 이전 토큰 무효화.
7. **명단 개인정보**: 학생 명단의 서버 전송·저장 금지(전 기능 클라이언트 완결). URL 인코딩은 명시적 고지 후에만. (예외: Notion 명렬표 불러오기는 이름이 연동 계층을 **통과만** 한다 — 서버 저장·캐시·로그 금지, 결과는 localStorage 프리셋에만.)

## 6. 코딩 규약

- **TS strict** 모드. 모든 외부 경계(URL 설정 문자열, Notion 응답, OAuth 응답, API 요청)는 zod로 파싱 후 사용.
- **설정 문자열**: zod 스키마(core) → JSON → lz-string → base64url → `?c=`. 디코딩 실패 시 **기본 설정 폴백 + 화면 내 안내 배지**(빈 화면 금지).
- **타이머**: 타임스탬프 기준 계산. setInterval 누적 금지(탭 비활성 드리프트 보정).
- **알고리즘**: 자리배정은 시드 기반 셔플 + 제약 충족. 제약 불능 시 명시적 실패 반환(무한루프 금지, 시도 상한 §11).
- **Notion 접근**: 속성은 반드시 매핑 테이블을 통해서만 접근(속성명 하드코딩 금지). 레이트리밋 ~3 req/s 준수, 읽기는 서버 캐시로 완충.
- **UI**: 이모지 금지 — SVG 아이콘(Lucide 단일 라이브러리)·컬러 닷 사용. 색상은 CSS 커스텀 프로퍼티로만(hex 하드코딩 금지). 다크모드 지원.

## 7. 검증 / 테스트 정책

- **core 순수함수는 Vitest 단위테스트 필수**: 설정 코덱 왕복(인코딩→디코딩 동일성), 집계(NaN 없음, 합계 = Σ행), 자리배정 제약 충족, 자동기록 멱등성.
- Notion 읽기 실패: 재시도 3회(지수 백오프) → 직전 캐시 표시 + 오프라인 배지.
- Notion 쓰기 실패: 재시도 3회 → localStorage 대기열 보관 + 재전송 버튼(**기록 유실 금지**).
- 연동 위젯 상태 전이: `ANONYMOUS → CONNECTED → (DEGRADED ⇄ CONNECTED) → REVOKED`. DEGRADED = 캐시 유지 + 쓰기 대기열, REVOKED = 재인가 유도 + 로컬 기능 지속.
- 종단 테스트: 실제 Notion 워크스페이스에서 온보딩→임베드→기록→집계 전 과정 확인.

## 8. 빌드 순서 (버티컬 슬라이스)

1. **core + 시계**: 설정 코덱(왕복 테스트) → 시계 위젯 → `/w/`·`/f/` 라우팅 → 정적 배포
2. **수업 도구**: 자리배정·추첨 core + 테스트 → 자리뽑기·랜덤뽑기·수업 타이머 UI → 명단 저장 규약
3. **뽀모도로**: 상태머신 core + 테스트 → UI + localStorage 기록
4. **서버 기반**: OAuth → 토큰 암호화 저장 → 위젯 토큰 발급/검증 → Notion 프록시 골격
5. **대시보드**: 템플릿 DB + 속성 매핑 → 읽기·집계·차트 → 기록 쓰기 + 실패 대기열 → 뽀모도로 자동기록
6. **빌더 홈**: 위젯 갤러리 + 설정 UI + URL 발급 + 온보딩
7. **종단 테스트**

## 9. 프론트 UI 표준

[DESIGN.md](DESIGN.md)의 Notion 디자인 시스템을 따른다. 핵심만:

- **따뜻한 중성색**: 텍스트 `rgba(0,0,0,0.95)`(순흑 금지), 웜 그레이 `#f6f5f4 / #615d59 / #a39e98`(블루그레이 금지)
- **위스퍼 보더** `1px solid rgba(0,0,0,0.1)`, 그림자는 다층 스택(개별 opacity ≤ 0.05)
- **액센트는 Notion Blue `#5c5c5c` 하나만**(CTA·링크·인터랙티브 전용)
- 타이포: Inter 계열, 디스플레이 크기에서 네거티브 자간(-2.125px@64px), 가중치 400/500/600/700
- 레이아웃: 8px 기반 스페이싱, 컴팩트·고밀도(참조 대비 ~70% 섹션 간격)
- 전체화면(`/f/`) 모드: 교실 뒤에서도 보이는 vw 기반 초대형 타이포
- **위젯 카드 = 파스텔 스티커 스타일**: 임베드 위젯은 16px 라운드 카드, 전 위젯 공통 `bg` 설정(default/charcoal/pink/mint/green/blue/purple/sand — `--pastel-*` 토큰). 카드 안에서는 `--bg`/`--fg` 등 토큰을 카드 색으로 재정의해 위젯 코드는 변수만 사용
- 다크모드 토큰·모션 토큰(150/220/360ms, 스프링·바운스 금지)은 DESIGN.md 참조

## 11. 정책 수치 (초기 기본값)

- 대시보드 읽기 캐시 TTL **30~60초** · Notion 재시도 **3회**(지수 백오프) · **~3 req/s** 준수
- 위젯 토큰 만료 **무기한**(재발급 시 이전 토큰 무효화)
- 자리뽑기 제약 충족 시도 상한 **1,000회**(초과 시 "조건 완화 필요" 반환)
- 명단 프리셋 localStorage 최대 **10개** · 자리배치 히스토리 **5회**
- 속성 매핑 필수 4종: `날짜`(date) · `분류`(select) · `시간(분)`(number) · `메모`(rich_text, 선택)
