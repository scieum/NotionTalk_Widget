# 교실 알고리즘 사양 (packages/core 순수함수)

공통 원칙: **시드 기반 결정론** — 같은 시드 + 같은 입력 = 같은 결과. `Math.random()` 직접 사용 금지, mulberry32 등 시드 PRNG를 core에 하나 두고 공유한다. 테스트 가능성이 목적.

## 자리배정 (`assignSeats`)

입력: 격자(행×열, 기본 5×6) + 비활성 칸 집합 + 학생 명단 + 제약 + 시드

제약 종류:
- **고정석**: 학생 X는 칸 (r,c) 고정
- **분리**: 학생 X, Y는 이웃(상하좌우 + 대각선) 금지

알고리즘:
1. 고정석 먼저 배치 (고정석끼리 충돌 → 즉시 실패 반환)
2. 나머지를 시드 셔플 후 배치, 분리 제약 위반 시 재셔플
3. 시도 상한 **1,000회** 초과 → `{ ok: false, reason: 'constraints-unsatisfiable' }` 반환 (UI: "조건 완화 필요"). **무한루프 금지.**

반환은 항상 명시적 결과 타입 `{ ok: true, seats } | { ok: false, reason }` — throw 금지.

검증 필수 케이스:
- 학생 수 > 가용 칸 수 → 명시적 실패
- 고정석이 비활성 칸을 가리킴 → 명시적 실패
- 분리 쌍이 물리적으로 충족 불능(예: 칸 2개에 분리 쌍) → 상한 후 실패
- 같은 시드 → 같은 배치 / 다른 시드 → (대체로) 다른 배치
- 결과에 중복 학생·중복 칸 없음, 비활성 칸 미사용

## 랜덤 추첨 (`drawStudents`)

입력: 명단 + 뽑을 인원 N + 중복 허용 여부 + 이미 뽑힌 집합 + 시드

- 중복 제외 모드: 남은 인원 < N이면 남은 전원 반환 + `exhausted: true` (에러 아님 — UI가 "명단 리셋" 제안)
- 중복 허용 모드: 항상 N명
- 순차 공개 애니메이션은 UI 몫 — core는 최종 결과 배열만 반환

## 히스토리

- 자리배치 결과는 localStorage에 최근 **5회** 저장 (이전 배치 다시 보기)
- 명단 프리셋은 최대 **10개** — 저장 규약은 widget-config 스킬 참조 (명단은 URL 금지 기본)
